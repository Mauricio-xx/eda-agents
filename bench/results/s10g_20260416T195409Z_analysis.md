# S10g LLM-sizing skill-injection A/B — analysis

TS: `20260416T195409Z`. Driver: `scripts/run_s10g_llm_sizing_ab.sh`.
Comparator output at `s10g_20260416T195409Z_comparison.md`.

## Headline

**POSITIVE signal — Pass@1 +10 % absolute.** First confirmation in
the whole validation arc that injecting skills into the LLM system
prompt actually changes outcomes, not just prompt size.

| dimension                    | skills ON (N=10) | skills OFF (N=10) | delta |
|------------------------------|------------------|-------------------|-------|
| Pass@1                       | 10/10 = 1.00     | 9/10 = 0.90       | **+0.10 abs** |
| mean Adc_dB                  | 59.81            | 34.32             | +25.49 dB |
| mean GBW_Hz                  | 2.20e6           | 1.76e6            | +0.44e6 |
| mean total_tokens (per call) | 4 141            | 364               | 11.4× |
| mean wall-time               | 2.9 s            | 1.9 s             | +1.0 s |

## Setup

- Task: `spec_llm_miller_ota_ihp_ab` (IHP SG13G2, Miller OTA),
  one-shot LLM sizing against audit thresholds `Adc ≥ 25 dB`
  and `GBW ≥ 500 kHz`.
- Backend: `google/gemini-2.5-flash` via OpenRouter, temperature 0.7
  (contrast with the deterministic S9 gap #8 task which uses
  temperature 0.0).
- Runs: 10 seeds per condition, conditions set via
  `EDA_AGENTS_INJECT_SKILLS={0,1}`.

## Per-seed raw

```
on  seed1..10: 10/10 PASS, params={gmid_input:12, gmid_load:10,
               L_input:5e-7, L_load:5e-7, Cc:5e-13}, Adc=59.8 dB
off seed1:     FAIL_AUDIT, {gmid_input:12, gmid_load:8,   L_load:7e-7}, Adc=22.5 dB  (below 25 dB spec)
off seed2:     PASS,       {gmid_input:15, gmid_load:10, L=5e-7},        Adc=36.3 dB
off seed3..7:  PASS same as seed2
off seed8:     PASS,       {gmid_input:12.5, gmid_load:10},               Adc=30.4 dB
off seed9,10:  PASS same as seed2
```

## Mechanism

With skills prepended, Gemini locks onto the gm/ID canonical
moderate-inversion pair (gmid_input=12 S/A, gmid_load=10 S/A,
L=0.5 µm, Cc=0.5 pF). Ten independent samples at temperature 0.7
produced the *same* params exactly — the methodology text in
`analog.miller_ota_design` + `analog.gmid_sizing` acts like a prior
that collapses the posterior to the textbook midpoint.

Without skills, Gemini still finds valid configurations 9 times out
of 10, but it leans toward `gmid_input=15` (weaker inversion,
slightly lower gm/current) and one of the ten samples wandered to
`gmid_load=8` with a longer `L_load=7e-7`. That combination drove
Adc to 22.5 dB — 2.5 dB below the 25 dB spec — and audit failed.
The failure is exactly the kind skills are supposed to prevent: a
confident-sounding but off-manifold proposal.

## Budget

Tokens ratio is 11.4× (4 141 vs 364 per call). That is larger than
the 2.16× observed on the digital counter bench because here the
user prompt is a short YAML spec while the skill bundle is
~6.5 k characters of methodology text. 4 141 tokens is still well
below the 12 k soft cap `render_relevant_skills` warns at — no
action needed.

Wall-time rose by ~1 s per call (2.9 s vs 1.9 s) — consistent with
the larger prompt's extra prefill time. Cost over the 20 runs was
≈ $0.01.

## Honest caveats

- **N=10 per arm** gives a directional signal, not statistical
  certainty. A Fisher's exact test on 10/10 vs 9/10 does not reach
  conventional significance. The **qualitative** pattern — skills
  collapse variance at the canonical design point — is the stronger
  evidence. A follow-on run at N=30+ would harden the result.
- **One-shot by construction**: `llm_spec_to_sizing_adapter` does
  not loop. That is the whole point of the experiment — we want to
  observe first-shot proposal quality — but it means we cannot
  distinguish "skills help the first proposal" from "skills help
  exploration". The digital counter-live bench covers the latter
  (and showed neither positive nor negative there; see
  `s10c_20260416T185823Z_analysis.md`).
- **Gemini-specific**: results would likely differ on Haiku-4.5,
  GPT-5, etc. The audit does not isolate which model features the
  skill text exploits.
- The audit is lax (25 dB DC gain, 500 kHz GBW). A tighter spec
  would probably widen the gap further since the OFF distribution
  includes marginal sizing.

## Decision

The S10c thesis — *methodology knowledge in skills improves first-shot
LLM proposals where domain knowledge matters* — has one clean
positive data point here. Combined with the no-regression evidence
from the digital counter A/B (`s10c_20260416T185823Z_analysis.md`)
and the SPICE + MCP wiring verified end-to-end, the default
`EDA_AGENTS_INJECT_SKILLS="1"` is now positively supported, not
just "not regressing". No default flip.
