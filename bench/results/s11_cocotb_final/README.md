## Wiring-probe journey (honest diagnostic path)

Three attempts before this final PASS:

1. **First attempt (`s11_cocotb_wiring_probe`)** — FAIL_AUDIT.
   Claude wrote `dut.en.value = 1` immediately after `await ReadOnly()`.
   cocotb silently drops writes during ReadOnly, so the DUT missed
   the enable pulse and the very first assertion "increment mismatch:
   got 0, expected 1" fired. Claude attempted a self-correction
   (dropped `ReadOnly`, added `FallingEdge`) but fell into the same
   race on a different edge. `make sim` kept failing.

2. **Second attempt (`s11_cocotb_wiring_probe_retry`)** — FAIL_AUDIT
   from a different bug. After adding an explicit warning in the
   `digital.cocotb_testbench` skill ("READONLY IS READ-ONLY: cocotb
   silently drops writes during ReadOnly"), Claude converged on the
   canonical FallingEdge-stimulus pattern and completed the flow
   SIGNOFF CLEAN. But the bench still reported FAIL_AUDIT: the
   `run_post_flow_gl_sim_check` helper hardcoded the testbench path
   to `tb/tb_<design>.v` (iverilog convention) and didn't know to
   skip when `tb/test_<design>.py` exists instead.

3. **Third attempt (this run `s11_cocotb_final`)** — PASS.
   `run_post_flow_gl_sim_check` now returns
   `{"skipped": True, "reason": "cocotb_tb_no_gl_sim_support"}`
   when a cocotb test file is present, and the adapter maps that to
   `sim_ok=True + gl_sim_skipped=1.0`. Flow ran, metrics matched
   the YAML's expected_metrics, gate went green.
