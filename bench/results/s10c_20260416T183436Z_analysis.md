## Headline

The automated comparator prints **GATE GREEN** but that verdict is
**trivial**: every LLM-backed digital run FAILed in both conditions
on the same LibreLane infra issue. The delta test reads
`0 - 0 = 0`, which satisfies the 5 % gate by construction while
measuring nothing about skill injection.

What this bench *does* confirm:

1. **Skill injection wiring reaches the LLM.** `total_tokens` per run
   is 3770 ON vs 1626 OFF — a deterministic **2.3× ratio**. The skill
   content prepended by `DigitalAutoresearchRunner._system_prompt()`
   shows up in the real prompt the model sees.
2. **The instrumentation patch works.** S10e captured `response.usage`
   cleanly; tokens surfaced in `metrics.total_tokens` on the
   BenchResult JSON.
3. **The sanity analog callables still pass** (`miller_ota` PASS on
   both sides). The SAR 11-bit ENOB task is marginal in isolation and
   flipped between ON and OFF in a single replicate (see below); not
   S10c-related.

What this bench **does not** confirm:

- That injecting skills produces better LLM proposals than not, in
  terms of Pass@1 or FoM. The LibreLane step crashed before any
  design could be hardened, so both conditions collapsed to
  `kept=0/2` and `best_fom=0.0`.

## Root cause of the digital FAIL

Both proposals in every run crashed at the same early LibreLane step:

```
Eval #1: crash -- ...osys-jsonheader/counter.h.json']) failed
```

`Yosys.JsonHeader` is an early pre-synthesis step that generates
`counter.h.json` for downstream tooling. The fact that it crashes for
every param combination (two different seeds per eval × three sweep
seeds = six independent crashes) points at infra / tool-chain
incompatibility between the LibreLane venv pinned at
`/home/montanares/git/librelane/.venv/bin/python` and the GF180
wafer-space fork at `/home/montanares/git/wafer-space-gf180mcu`, not
at anything the LLM or skill injection proposed. Every crash status
was a `"crash"` row in the per-eval TSV, i.e. the autoresearch loop
saw them as LibreLane-owned errors, not as invalid params.

## Why the 3 seeds collapsed to 1 data point

`total_tokens` is byte-identical across the three seeds within each
condition (3770 / 3770 / 3770 ON; 1626 / 1626 / 1626 OFF). Despite
`temperature=0.7` in `DigitalAutoresearchRunner._propose_params`,
either:

- OpenRouter cached responses for identical prompts across the three
  rapid re-runs, OR
- The model stayed in its deterministic-enough region for the tiny
  budget-2 prompts at this temperature.

Either way, the effective sample size is N=1 per condition even
though we paid for N=3. That's another reason to call this bench
infra-limited, not conclusive.

## Analog sanity

`miller_ota` PASS on both sides, delta 0 — the deterministic callable
adapter is unaffected, confirming S10c didn't introduce collateral
damage to the non-LLM path.

`sar_adc_11bit` flipped: `PASS` with skills OFF, `FAIL_AUDIT`
(`SNDR_dBc=16.55` OUT of range) with skills ON, on single replicates.
This task does not invoke the LLM at all — both conditions run the
exact same `run_sar11_enob_measurement` callable. The flip is the
pre-existing near-threshold flakiness acknowledged in
`project_s9_gap_closure_commitment.md` (gap #6b raised the anchor to
the measured ceiling). Not an S10c signal.

## Verdict

**INCONCLUSIVE.** The one observation we can defend from this run:

- Skills are reaching the LLM (token delta).
- The wiring is live (patch + runner + adapter round-trip clean).
- No collateral damage to callable analog paths.

We do **not** yet have empirical evidence that skill injection
improves Pass@1 or FoM. To get that, the yosys-jsonheader crash
needs to be fixed first, or the A/B moved to a different LLM-backed
task that can actually run LibreLane to completion (e.g. the
`e2e_digital_counter_gf180` hardening task, at 45 min/run × 6).
