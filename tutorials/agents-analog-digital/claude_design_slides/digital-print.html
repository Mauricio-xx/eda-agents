Where the agents sit
The seven phases are LibreLane's responsibility. The digital agents sit on top and decide when to re-run what with what knobs. They never invoke Yosys, OpenROAD, Magic, KLayout, or Netgen directly — every action goes through a LibreLane step.
Mauricio Montanares · IHPPart 01 · problem
Part 1 · problem6 / 42

LibreLane is an orchestrator, not an EDA tool

Single command: librelane config.yaml. Under the hood, six independent open-source projects cooperate.

  • Yosys — RTL → gate-level netlist.
  • OpenROAD — floorplan, PDN, placement, CTS, routing.
  • Magic — DRC + GDS streamout.
  • KLayout — second DRC engine + XOR cross-check.
  • Netgen — LVS.
  • OpenSTA — static timing analysis.
Agent implication
When a flow fails, the fix is in one of the six tools' logs — not in LibreLane itself. Agents diagnose at the tool layer.

Part 1 · problem7 / 42

Cost asymmetry — why digital autoresearch is discrete

Per LibreLane run
~ minutes
Small counter: 1–2 min. Chipathon padring: 35–45 min. Magic streamout can dominate on IHP.
Per LLM proposal
~ cents – $1
Model-dependent. Gemini Flash « Sonnet « Opus. OpenCode multi-provider opens the cheap end.
Search-strategy implication
Budget per run: a handful of evaluations. DigitalAutoresearchRunner defaults to budget=5. Design space is discrete — density in {60, 65, 70}, clock in {50, 40, 30} ns, PDN pitch in {4, 6, 8} — with nearest-value snapping on proposals.

synthesis_engineer

signoff_checker

DigitalAutoresearch

Greedy search over LibreLane knobs. Where OpenCode plugs in as a first-class backend.

Mauricio Montanares · IHPPart 04 · autoresearch
Part 4 · autoresearch19 / 42

DigitalAutoresearchRunner

  • Same greedy shape as analog — Propose → Map → Evaluate → Score → Log → Iterate.
  • Discrete space. density ∈ {60, 65, 70}, clock ∈ {50, 40, 30} ns, PDN pitch ∈ {4, 6, 8}.
  • Budget 5 by default. Each eval = full LibreLane run.
  • FoM: weighted timing (WNS), area, power.
Constructor · backends accepted
DigitalAutoresearchRunner( design=design, backend="opencode", # adk | cc_cli | litellm | opencode budget=5, )

Four backends, one runner

adk
Google ADK runner. 5-agent tree, locked-down tool sets. CI-friendly.
cc_cli
claude --print --output-format json. Anthropic subscription.
litellm
Plain LiteLLM API call. No CLI, no agent hierarchy. Deterministic + cheap.
opencode
opencode run --format json. Multi-provider: Anthropic, OpenAI, Gemini, Z.AI, Ollama.
Backend choice is a string

Caveat — ProjectManager vs OpenCode

ProjectManager today
Accepts backend="adk" or backend="cc_cli". Raises ValueError on anything else. This is deliberate phasing, not an oversight.
OpenCode path today
End-to-end RTL-to-GDS via OpenCode:
opencode --agent gf180-docker-digital in the TUI.
Knob-search autoresearch via OpenCode:
DigitalAutoresearchRunner(backend="opencode") in Python.
Wiring OpenCode into the ProjectManager multi-agent tree is planned; see docs/s12_handoffs/s12d_opencode_wrapper.md.

GenericDesign — what it wraps

  • Takes a LibreLane config.yaml path and an optional pdk_root.
  • Derives the 13 DigitalDesign methods the agents expect: project_name, specs_description, fom_description, validate_clone, etc.
  • Supports FoM weight overrides without subclassing.
  • Source: src/eda_agents/core/designs/generic.py.
Instantiation
from eda_agents.core.designs.generic import GenericDesign design = GenericDesign( config_path="./counter_project/config.yaml", pdk_root=None, # falls back to $PDK_ROOT pdk_config="gf180mcu", fom_weights={"timing_w": 1.0, "area_w": 0.5, "power_w": 0.3}, )

Generic → ProjectManager → flow

import asyncio from eda_agents.core.designs.generic import GenericDesign from eda_agents.agents.digital_adk_agents import ProjectManager design = GenericDesign(config_path="./counter_project/config.yaml", pdk_config="gf180mcu") pm = ProjectManager(design=design, model="google/gemini-3-flash-preview", backend="cc_cli", max_budget_usd=1.00) # Dry-run first (safe, no Docker, no LLM) result = asyncio.run(pm.run(Path("./results"), dry_run=True)) # Real run when ready result = asyncio.run(pm.run(Path("./results")))
This is the entire glue. No registry to edit, no bespoke Python class. Works for any LibreLane-compatible design.

When to subclass DigitalDesign explicitly

  • Design fetch hooks — nix-shell, submodule update, git-lfs, custom RTL generators.
  • Pre-flow RTL prep — Verilog generation from templates, parameter sweeps.
  • Post-flow metric extraction — custom metrics beyond metrics.csv.
  • Multi-macro designs — when one config does not cover the whole project.
Shipped examples
fazyrv_hachure.py — GF180 RISC-V SoC, uses nix-shell.
systolic_mac_dft.py — CI fixture, simpler.
Both at src/eda_agents/core/designs/. ~150 lines each — explicit is small.

Safety — notebooks default to rehearse

  • Every RUN_* flag defaults to False.
  • Opening a notebook runs no Docker, no LLM, no cost.
  • Each flip is a deliberate decision. A teaching session can step through the whole story without committing to a paid run.
  • Identical policy on both analog and digital notebooks.
Budget guard
ProjectManager(max_budget_usd=1.00) is the hard cap for cc_cli runs. Exceed it and the flow stops cleanly with a surfaced verdict.
